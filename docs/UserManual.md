# TRPG Studio ユーザーマニュアル

対象バージョン: Windowsローカル版 1.0.0

TRPG Studio は、TRPGシナリオをページ単位で管理しながら、本文執筆、NPC管理、Markdown風プレビュー、シーンテンプレート挿入、画像埋め込み、HTML/Markdown出力を行うためのデスクトップエディタです。

## 1. 起動と基本構成

### 1.1 起動方法

開発環境では以下のコマンドを使用します。

```bash
npm run electron:dev
```

Web表示のみを確認する場合は以下を使用します。

```bash
npm run start
```

製品ビルドは以下です。

```bash
npm run build
```

配布用パッケージを作成する場合は以下を使用します。

```bash
npm run make
```

### 1.2 画面構成

アプリは大きく2つの画面で構成されています。

- シナリオ一覧画面: シナリオの作成、選択、複製、削除、並べ替え、インポート、設定を行います。
- エディタ画面: 選択したシナリオの本文、ページ、NPC、画像、出力、表示設定を編集します。

## 2. シナリオ一覧画面

### 2.1 新規シナリオを作成する

1. 画面右上または空状態画面の「新規作成」を選択します。
2. シナリオ名を入力します。
3. 「作成する」を選択します。

作成後、自動的にエディタ画面へ移動します。新規シナリオには初期キャラクターとして `GM` と `PL1` が登録されます。

### 2.2 シナリオを開く

一覧に表示されているシナリオカードまたは行を選択すると、そのシナリオのエディタ画面へ移動します。

### 2.3 表示形式を切り替える

一覧画面右上の表示切替ボタンで、以下を切り替えられます。

- グリッド表示: カード形式でシナリオを並べます。
- リスト表示: 横長の一覧形式でシナリオを並べます。

### 2.4 シナリオを並べ替える

シナリオカードまたは行をドラッグ&ドロップすると、表示順を変更できます。

保存先フォルダを設定している場合、並び順は各シナリオJSON内の `sortOrder` として保存されます。ローカルストレージ利用時も同様に順序情報が保存されます。

### 2.5 シナリオを複製する

1. 対象シナリオの複製ボタンを選択します。
2. 確認ダイアログで「複製する」を選択します。

複製されたシナリオは新しいIDを持ち、一覧の先頭に追加されます。

### 2.6 シナリオを削除する

1. 対象シナリオの削除ボタンを選択します。
2. 確認ダイアログで「削除する」を選択します。

削除操作は取り消せません。保存先フォルダを使用している場合、同期時に対応するJSONファイルも削除されます。

### 2.7 Markdownまたはテキストファイルをインポートする

1. 一覧画面のインポートボタンを選択します。
2. `.md` または `.txt` ファイルを選択します。

インポート時の仕様は以下です。

- ファイル名から拡張子 `.md` を除いた文字列がシナリオ名になります。
- `# 見出し` で始まるページは、その見出しをページタイトルとして扱います。
- ページ区切りは空行付きの `---` です。
- `[キャラクター名] セリフ` 形式の行からキャラクター名を抽出し、NPC/登場人物リストに追加します。
- `data:image/...` 形式の埋め込み画像は内部画像IDへ変換されます。

## 3. エディタ画面

### 3.1 ヘッダー

エディタ上部には以下の操作があります。

- ページサイドバーの表示/非表示
- 一覧画面へ戻る
- シナリオ名の変更
- 保存状態の表示
- 表示モード切替
- 設定
- ライト/ダーク切替
- エクスポート
- 現在ページの内容クリア

シナリオ名を選択すると、名前変更ダイアログが開きます。

### 3.2 保存状態

本文、ページ、キャラクター、画像の変更は自動保存されます。

保存状態はシナリオ名の横に表示されます。

- `編集...`: 変更検知後、保存待ちです。
- `保存しました`: 直近の保存が完了しました。
- `保存済み`: 保存済みの安定状態です。

一覧へ戻るときは即時保存が実行されます。

### 3.3 表示モード

エディタは3つの表示モードを持ちます。

- 分割表示: 左に本文エディタ、右にプレビューを表示します。
- 編集表示: 本文エディタのみを表示します。
- プレビュー表示: プレビューのみを表示します。

分割表示では、本文側のカーソル位置やスクロールに合わせてプレビュー側の位置が同期されます。

## 4. ページ管理

### 4.1 ページサイドバー

左サイドバーにはシナリオ内のページ一覧が表示されます。

ページを選択すると、そのページを編集対象にします。

### 4.2 ページを追加する

1. ページサイドバー下部の「新規ページを追加」を選択します。
2. ページ名を入力します。
3. 「追加する」を選択します。

ページ名を空にした場合は `新規ページ N` 形式の名前が使われます。

### 4.3 ページ名を変更する

エディタ上部のページタイトル部分を選択すると、ページ名変更ダイアログが開きます。

### 4.4 ページを削除する

ページサイドバー内の削除ボタンを選択すると確認ダイアログが表示されます。確定するとページが削除されます。

削除したページは復元できません。

### 4.5 ページを並べ替える

ページサイドバー内でページをドラッグ&ドロップすると、ページ順を変更できます。

### 4.6 アウトラインを使う

ページ本文に `# 見出し` または `## 見出し` が含まれる場合、ページ項目の展開ボタンからアウトラインを表示できます。

アウトライン項目を選択すると、該当見出し位置へ本文カーソルが移動します。

## 5. 本文エディタ

### 5.1 基本入力

本文エディタにはシナリオ本文を自由に入力できます。右側のプレビューには、TRPG Studio独自のMarkdown風レンダリングが反映されます。

### 5.2 対応記法

#### 見出し

```markdown
# 大見出し
## 中見出し
### 小見出し
```

#### 太字

```markdown
**重要な情報**
```

#### 斜体

```markdown
*強調*
```

#### 取り消し線

```markdown
~~削除された情報~~
```

#### 区切り線

```markdown
---
```

#### 箇条書き

```markdown
- 項目1
- 項目2
```

#### セリフ

```markdown
[GM] 扉の向こうから足音が聞こえる。
[NPC名] よく来たな、冒険者たち。
```

`[名前] 本文` の形式で書くと、プレビューではTRPGのセリフ形式として表示されます。

#### 情報ブロック

```markdown
> [探索情報] 机の引き出しには古い鍵が入っている。
> 続きの情報も同じブロックに含められる。
```

`>` で始まる行は情報ブロックとして表示されます。情報ブロックにはコピー用ボタンが表示され、プレビュー上で内容をコピーできます。

Enterキーで情報ブロック行を改行すると、自動で次行にも `> ` が挿入されます。空の `>` 行でEnterすると、情報ブロックを終了できます。

#### 補足ブロック

```markdown
||この情報はGM向けの補足です。||
```

#### 画像

画像挿入機能を使うと、本文には以下の形式で画像トークンが挿入されます。

```markdown
![画像名](image:画像ID)
```

画像データはシナリオデータ内の `images` にBase64形式で保存されます。

画像サイズは5MB以下にしてください。

### 5.3 ツールバー

ページタイトル下のツールバーから、以下を挿入または実行できます。

- シーンテンプレート
- セリフ用キャラクター選択
- NPCリスト表示切替
- 見出し
- 太字
- 取り消し線
- 情報ブロック
- 補足ブロック
- 画像挿入
- 検索
- 置換

## 6. シーンテンプレート

ツールバーの「シーン」ボタン、またはショートカットからシーン追加ダイアログを開けます。

入力項目は以下です。

- シーンタイトル
- シーン番号
- プレイヤー
- 登場NPC

シーンタイトルとシーン番号は必須です。

挿入されるテンプレートは概ね以下の構造です。

```markdown
## ● シーン1：シーンタイトル（PC プレイヤー）
### ◆ 解説

### ▼ 描写&セリフ
**[登場NPC：NPC名]**

### ▼ 結末
```

プレイヤーまたは登場NPCが未入力の場合は確認ダイアログが表示されます。

## 7. NPC/登場人物管理

### 7.1 NPCサイドバーを開く

ツールバーの `NPC` ボタン、またはショートカットで右サイドバーを開きます。

### 7.2 NPCを追加する

右サイドバー上部の入力欄に名前を入力し、追加ボタンを押します。

同じ名前が既に存在する場合は重複追加されません。

### 7.3 NPC名を変更する

NPCリスト内の名前を選択すると編集状態になります。名前を変更し、Enterまたはフォーカスアウトで確定します。

既存の別NPC名と重複する名前には変更されません。

### 7.4 NPCを削除する

NPC行の削除ボタンを選択すると確認ダイアログが表示されます。確定するとリストから削除されます。

本文内に既に書かれた `[NPC名]` の文字列は自動変更されません。

### 7.5 セリフ挿入

ツールバーの「セリフ」ボタンでキャラクター選択ポップアップを開けます。

キャラクターを選ぶと、本文カーソル位置に以下の形式で挿入されます。

```markdown
[キャラクター名] 
```

## 8. 検索と置換

### 8.1 検索

検索ウィジェットでは、現在ページの本文内を検索できます。

- Enter: 次の一致へ移動
- Shift+Enter: 前の一致へ移動
- Esc: 検索ウィジェットを閉じる

### 8.2 置換

置換モードでは、検索語と置換語を入力できます。

- 置換: 現在選択中の一致箇所を置換します。
- すべて置換: 現在ページ内の一致箇所をすべて置換します。

検索と置換は現在ページの本文に対して行われます。

## 9. エクスポート

ヘッダー右側のエクスポートメニューから以下を出力できます。

### 9.1 Markdownで保存

シナリオ全ページを1つのMarkdownファイルとして保存します。

仕様:

- 各ページは `# ページタイトル` から始まります。
- ページ間は `---` で区切られます。
- 内部画像トークンはBase64画像に展開されます。

### 9.2 現在ページをHTMLで保存

現在開いているページだけをHTMLとして保存します。

仕様:

- プレビューと同じレンダリング結果をHTML化します。
- 見出しから目次を生成します。
- 印刷時のレイアウトを考慮したCSSを含みます。

### 9.3 全シナリオをHTMLで保存

シナリオ内の全ページを1つのHTMLとして保存します。

仕様:

- 各ページはHTML内のセクションとして出力されます。
- ページタイトルとページ内見出しから目次を生成します。
- 印刷時はページごとに改ページされます。

## 10. 設定

設定画面には以下のタブがあります。

- 一般
- エディタ
- ショートカット

一覧画面から開いた場合、エディタ固有設定の一部は表示されないことがあります。

### 10.1 テーマ設定

利用できるテーマは以下です。

- defaultテーマ: 装飾的な背景とガラス調UIを使う標準テーマです。
- simpleテーマ: 白黒2トーンを基調とした、フラットで簡素なテーマです。

### 10.2 カラーモード

以下を切り替えられます。

- ライト
- ダーク

### 10.3 データ保存先フォルダ

保存先フォルダを絶対パスで指定できます。

例:

```text
C:\TRPG_Data
```

または

```text
/home/user/TRPG_Data
```

保存先フォルダを指定すると、以下の構成でデータが保存されます。

```text
指定フォルダ/
  scenario/
    シナリオID.json
  settings/
    settings.json
```

保存先フォルダを空にすると、ブラウザ/アプリ内のローカルストレージに保存されます。

注意:

- 保存先は絶対パスで指定してください。
- 存在しないフォルダは設定画面では指定できません。事前に作成してください。
- 指定したパスがフォルダでない場合は保存できません。
- Electronデスクトップ版でのみフォルダ保存が利用できます。

### 10.4 エディタ設定

エディタタブでは以下を変更できます。

- フォントファミリー
- フォントサイズ

フォント設定は本文エディタとプレビュー本文に反映されます。simpleテーマでもエディタ本文部分には設定フォントが反映されます。

### 10.5 ショートカット設定

ショートカットタブでは、各操作のキー割り当てを変更できます。

変更方法:

1. 変更したい操作のキー表示ボタンを選択します。
2. 割り当てたいキーを押します。
3. 設定を保存します。

「初期設定に戻す」を選択すると、ショートカット設定が初期値に戻ります。

## 12. データ仕様

### 12.1 シナリオデータ

シナリオは概ね以下の形式で保存されます。

```json
{
  "id": "scenario-id",
  "title": "シナリオ名",
  "pages": [
    {
      "id": "page-id",
      "title": "ページ名",
      "content": "本文"
    }
  ],
  "characters": ["GM", "PL1"],
  "images": {
    "image-id": "data:image/png;base64,..."
  },
  "lastModified": 1710000000000,
  "sortOrder": 0
}
```

### 12.2 保存方式

保存方式は2種類あります。

#### フォルダ保存

設定画面で保存先フォルダを指定した場合、シナリオごとにJSONファイルとして保存されます。

#### ローカルストレージ保存

保存先フォルダを指定していない場合、アプリ内のローカルストレージへ保存されます。

### 12.3 緊急バックアップ

フォルダ保存中にドライブ切断などで保存に失敗した場合、アプリはローカルストレージに緊急バックアップを保存します。

対象キー:

- `trpg_scenarios_emergency_backup`
- `trpg_shortcuts_emergency_backup`

## 13. トラブルシューティング

### 13.1 保存先フォルダが見つからない

前回指定した保存先フォルダが存在しない場合、起動時に確認ダイアログが表示されます。

選択肢:

- 新規フォルダを作成: 同じ場所にフォルダを作成し、フォルダ保存を継続します。
- キャンセル: 一時的にローカルストレージ保存で起動します。

### 13.2 フォルダ保存ができない

以下を確認してください。

- 絶対パスで指定しているか
- フォルダが存在するか
- ファイルではなくフォルダを指定しているか
- アプリに書き込み権限があるか
- 外付けドライブやネットワークドライブが接続されているか

### 13.3 画像が挿入できない

画像ファイルが5MBを超えている場合は挿入できません。画像を圧縮してから再度挿入してください。

### 13.4 ショートカットが効かない

以下の場合、ショートカットは実行されません。

- 入力欄にフォーカスがある
- キャラクター選択、シーン追加、ページ追加、削除確認などのモーダルが開いている
- 設定したキーが別操作と競合している

### 13.5 プレビューの表示が想定と違う

TRPG StudioのMarkdown処理は独自実装です。一般的なMarkdownライブラリとは一部挙動が異なります。

特に以下に注意してください。

- セリフは行頭の `[名前] 本文` の形式で認識されます。
- 情報ブロックは `>` の後に空白が必要です。
- ページ区切りとして扱われる `---` はインポート時のみです。

## 14. 運用上の推奨

- 長期的に使う場合は、設定画面で保存先フォルダを指定してください。
- 保存先フォルダはクラウド同期対象にしても構いませんが、同時に複数端末で編集しないでください。
- 画像を大量に埋め込むとJSONファイルが大きくなります。必要に応じて画像サイズを圧縮してください。
- 大きな編集前にはMarkdownまたはHTMLでエクスポートしてバックアップしてください。

## 15. 開発者向けメモ

このアプリはElectron + Vite + React + Tailwind CSSで構成されています。

主要ファイル:

- `main.js`: Electronメインプロセス
- `src/App.jsx`: アプリ全体の状態と画面切替
- `src/hooks/useScenarioManager.js`: シナリオ一覧と永続化管理
- `src/components/ScenarioDashboard.jsx`: シナリオ一覧画面
- `src/components/trpg_editor/TRPGEditor.jsx`: エディタ画面の統合
- `src/components/trpg_editor/EditorArea.jsx`: 本文入力、検索置換、プレビュー
- `src/utils/markdownParser.js`: 独自Markdown風パーサ
- `src/utils/scenarioStorage.js`: シナリオ保存/読み込み
- `src/utils/scenarioImport.js`: Markdown/TXTインポート
